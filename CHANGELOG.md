# 변경 이력 (Changelog)

이 프로젝트의 모든 주요 변경 사항은 이 문서에 기록됩니다.
형식은 [Keep a Changelog](https://keepachangelog.com/ko/1.0.0/)를 따릅니다.

## [1.3.1] - 2026-07-05

멀티세션 UI 핫픽스입니다. 세션이 4개 이상일 때 드러난 두 가지 문제를 바로잡았습니다.

### 수정됨
- **"+N" 칩을 눌러도 대시보드가 뜨자마자 사라지던 문제**: 오버플로우 칩이 마우스 누름(`MouseLeftButtonDown`) 도중에 대시보드 팝업을 열면서, 같은 클릭의 마우스 뗌이 "바깥 클릭"으로 인식돼 팝업이 즉시 닫혔습니다. 팝업 열기를 입력 처리가 끝난 뒤로 미뤄(디스패처 지연) 정상적으로 열리도록 했습니다.
- **세션 칩 스트립 양 끝이 잘리던 문제**: 세션 칩 3개 + "+N" 칩의 총 너비가 창 폭(300px)을 넘겨, 가운데 정렬된 스트립의 양쪽 끝이 잘렸습니다. 기본 창 폭을 340px로 넓히고(캐릭터는 그대로 가운데 정렬) 칩 라벨 최대 폭을 살짝 줄여, 세션이 많아도 잘림 없이 표시됩니다.

## [1.3.0] - 2026-07-05

**멀티세션 인식** 릴리스입니다. 여러 Claude Code 세션을 동시에 돌려도 위젯이 각각을 구분해 보여주고, 세션별 비용·시간과 시스템 자원을 한눈에 보는 대시보드를 더했습니다.

### 추가됨
- **멀티세션 인식 + 세션 칩 스트립**: 동시에 실행 중인 Claude Code 세션을 훅 페이로드의 `session_id`로 구분해 추적합니다. 캐릭터 위에 세션별 칩(프로젝트명 + 상태 점)이 나열되고, 대표 캐릭터는 전체 세션 중 **가장 급한 상태(권한 대기 > 작업 중 > 유휴)** 를 표시합니다. 이전에는 여러 세션의 이벤트가 뒤섞여 말풍선·배지가 깜빡였는데, 이 문제가 사라졌습니다. 칩을 클릭하면 해당 세션의 상세(프로젝트·상태·경과)를 말풍선으로 보여줍니다.
- **세션 대시보드(HUD)**: 상태 점 클릭 · 세션 스트립의 "+N" 칩 · 우클릭 메뉴 **"세션 대시보드 보기"** 로 여는 카드형 패널입니다. 세션별로 상태·경과 시간·예상 비용을 나열하고, 하단에 실행 중인 Claude 프로세스의 **CPU·RAM 합계** 와 오늘/누적 코딩 시간을 함께 표시합니다.
  - **예상 비용**: 각 세션의 transcript(JSONL)를 증분 파싱해 토큰 사용량을 합산하고 대략적인 비용을 추정합니다(정확한 과금액이 아닌 참고용 "예상" 값). 시스템 자원은 세션별이 아니라 전체 Claude 프로세스 합계로 보여줍니다.
- **상태별 표정/연출**: 세션이 권한을 기다리면 캐릭터가 **걱정하는 표정**(눈썹 + 입)을 짓고, 작업을 마치면 머리 위 스파클이 **반짝 축하** 하며, 오래 유휴 상태면 눈을 감고 **Zzz 하며 졸기** 시작합니다.

### 변경됨
- **훅 우선 상태 감지**: 세션이 하나라도 추적되면 위젯 상태를 훅 이벤트(세션 레지스트리)에서 직접 도출하고, 프로세스 스캔은 폴백으로만 씁니다. 세션이 활성이고 대시보드가 닫혀 있을 때는 프로세스 폴링 주기를 2초 → 6초로 늦춰 부담을 줄였습니다.
- **상태 점 색상 + 동작**: 기존 회색(대기)/초록(작업)/파랑(업데이트)에 **앰버(확인 대기)** 가 추가됐습니다. 상태 점 클릭은 "오늘의 기록 말풍선"에서 **세션 대시보드 열기** 로 바뀌었습니다("오늘의 기록 보기"는 우클릭 메뉴에 그대로 있습니다).
- **창 레이아웃**: 세션 칩 스트립 공간을 위해 창 폭을 넓히고 캐릭터를 가운데 정렬했습니다.
- **내부 구조**: 프로세스 감지 로직을 `ClaudeProcessScanner` 서비스로 분리하고, 세션 상태(`SessionRegistry`)·transcript 비용(`TranscriptUsageReader`)·시스템 자원(`SystemUsageSampler`) 샘플링을 각각 별도 서비스로 나눴습니다.

## [1.2.0] - 2026-07-02

**CLI 연동 심화** 릴리스입니다. Claude Code와의 연동을 "알림을 받는" 수준에서 "지금 뭘 하는지 실시간으로 아는" 수준으로 끌어올렸습니다.

### 추가됨
- **툴 인식(작업 상세 표시)**: `PreToolUse` 훅을 새로 등록해, Claude가 지금 어떤 툴을 쓰는지 위젯 옆에 **활동 아이콘 배지**(Segoe MDL2 아이콘)와 짧은 말풍선으로 보여줍니다. Read/NotebookRead → "파일 읽는 중…", Edit/Write/MultiEdit/NotebookEdit → "코드 쓰는 중…", Bash → "명령 실행 중…", Glob → "파일 찾는 중…", Grep → "코드 검색 중…", WebFetch/WebSearch → "웹 읽는 중…"/"웹 검색 중…", Task → "에이전트 실행 중…", TodoWrite → "할 일 정리 중…", `mcp__*` → "도구 사용 중…", 그 외 → "작업 중…". 활동 배지는 툴 이벤트가 ~6초간 없으면 자동으로 사라지고, 말풍선은 깜빡임 방지를 위해 최대 ~4초에 한 번만 갱신됩니다. 우클릭 메뉴에 새 토글 **"작업 상세 표시 (툴 인식)"**(기본 켜짐)을 추가했고, `settings.json`에는 `ToolAwareness`로 저장됩니다.
- **권한/알림 강화**: `Notification` 이벤트(예: 권한 확인 대기)를 받으면 가재가 좌우로 흔들리고 말풍선을 띄우는 것에 더해 **트레이 풍선 알림(Windows 토스트)** 도 함께 띄워, 위젯이 숨겨져 있거나 다른 모니터에 있어도 놓치지 않도록 했습니다. 위젯이 숨겨져 있었다면 자동으로 다시 나타납니다.
- **프롬프트/세션 반응**: `UserPromptSubmit` → 통통 튀며 "요청 받았어요!", `SessionStart` → "세션 시작! 함께 코딩해요", `SessionEnd` → "세션 종료. 수고했어요!" 말풍선을 표시합니다.
- **휴식 넛지(뽀모도로)**: 작업이 50분 이상 끊김 없이 이어지면 "50분째 열일 중! 잠깐 쉬어요"라고 살짝 알려줍니다(별도 훅 없이 클라이언트 측 타이머로 동작).

### 변경됨
- **훅 등록 확장**: **Claude CLI 알림 연동** 토글이 이제 `Notification`/`Stop`/`SubagentStop`/`UserPromptSubmit`/`SessionStart`/`SessionEnd`(기본 이벤트)에 더해 `PreToolUse`(툴 이벤트)까지 `~/.claude/settings.json`에 함께 등록합니다. `PreToolUse`는 툴 호출마다 한 번씩 발생합니다.
- **경량 헤드리스 진입점**: `--hook`/`--notify` 실행 경로가 WPF를 전혀 로드하지 않는 전용 `Program.Main`을 거치도록 바뀌어, 훅 1회 호출 비용이 대략 절반(~230ms → ~120ms)으로 줄었습니다. 덕분에 툴 호출마다 실행되는 `PreToolUse` 훅도 부담 없이 쓸 수 있습니다.
- **구조화된 이벤트**: 인박스가 이제 일반 텍스트 대신 JSON `CliEvent` 객체(`{ "Kind": ..., "Tool": ..., "Text": ... }`, `Kind` ∈ `text`/`tool`/`notify`/`prompt`/`stop`/`substop`/`session`)를 저장합니다. `--notify "메시지"`는 여전히 그대로 동작하며(텍스트 이벤트로 기록), 과거 형식의 순수 텍스트 인박스 파일도 방어적으로 계속 인식합니다.

## [1.1.0] - 2026-07-02

대규모 고도화 릴리스입니다. 캐릭터 리디자인부터 자동 업데이트, Claude Code 알림 연동까지 위젯의 핵심 경험을 새로 다졌습니다.

### 추가됨
- **가재(게) 캐릭터 리디자인**: 기존의 단순한 주황색 덩어리 대신, 집게발·더듬이·부채 모양 꼬리·걸어다니는 다리를 갖춘 가재/게 모티브 캐릭터로 전면 교체. 더듬이 사이에 금빛 Claude 스파클(✦) 추가.
- **테마 5종**: 클로드(주황) · 바다(파랑) · 숲(초록) · 포도(보라) · 자정(다크) 중 우클릭 메뉴에서 즉시 선택 가능.
- **말풍선 시스템**: 작업 시작/완료, 인사 등 다양한 순간에 말풍선으로 말을 거는 기능. 우클릭 메뉴에서 on/off 가능.
- **시스템 트레이 아이콘**: 더블클릭으로 위젯 보이기/숨기기, 우클릭으로 보이기/숨기기·업데이트 확인·종료 메뉴 제공. 작업 표시줄 버튼은 노출하지 않음.
- **오늘의 기록(사용 통계)**: 오늘/누적 코딩 시간과 세션 수를 집계해 말풍선으로 확인 가능. `%APPDATA%\ClaudeWidget\settings.json`에 영속 저장.
- **Windows 시작 시 자동 실행**: 관리자 권한 없이 사용자 단위 레지스트리 Run 키를 통한 자동 시작 지원.
- **설정 영속화**: 테마, 크기, 위치, 항상 위에 표시, 말풍선 on/off, 자동 업데이트 on/off 등 모든 사용자 설정이 재시작 후에도 유지됨.
- **GitHub Release 기반 자동 업데이트**: 실행 시 및 6시간마다 GitHub Releases API를 확인해 새 버전을 감지하고, 클릭 한 번으로 설치 파일을 내려받아 조용히(`/SILENT`) 설치·재실행. 자세한 내용은 [docs/AUTO_UPDATE.md](docs/AUTO_UPDATE.md) 참고.
- **Claude CLI 알림 연동**: 위젯이 Claude Code의 훅(hook) 핸들러 역할을 겸해, `Notification`/`Stop`/`SubagentStop` 이벤트를 말풍선 메시지로 표시. `--hook`, `--notify` 진입점 제공. 자세한 내용은 [docs/CLI_INTEGRATION.md](docs/CLI_INTEGRATION.md) 참고.
- **앱 아이콘**: 트레이·바로가기·설치 프로그램에 사용되는 전용 앱 아이콘 추가.

## [1.0.0] - 최초 배포

- Claude Code 실행을 감지해 함께 코딩하는 기본 개념의 데스크톱 펫 최초 버전.
- 단순한 주황색 블롭(blob) 형태의 캐릭터로, Claude Code CLI가 실행 중이면 타이핑 애니메이션을 보여주고 그렇지 않으면 대기 상태를 유지.
